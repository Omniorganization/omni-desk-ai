# OmniDesk 1.11.1 iOS Real-Device Evidence Hardening

## Purpose

This release moves the project from source-level iOS readiness toward auditable real-device readiness.

It does not fabricate iOS test evidence. Instead, it adds a strict import path, JSON templates, and a validator for real iOS smoke-test outputs.

## Added

- iOS real-device evidence import guide.
- iOS native build evidence template.
- iOS signed IPA evidence template.
- APNS / Firebase APNS live delivery evidence template.
- `scripts/import_ios_real_device_evidence.py` for fail-closed validation and normalized copy into `release/external-evidence`.

## Required real evidence before customer-distribution GA

- `release/external-evidence/native-build/flutter-ios-release.json`
- `release/external-evidence/signed-artifacts/ios-signed-ipa.json`
- `release/external-evidence/push/apns-live-delivery.json`

## Validation command

```bash
python scripts/import_ios_real_device_evidence.py \
  --raw-dir /path/to/private-ios-evidence \
  --copy \
  --write-report release/ios-real-device-evidence-import-report.json

python scripts/check_external_ga_evidence.py .
```

## Security note

Only hashes and evidence metadata should be committed. Raw UDID, certificates, private keys, provisioning profile contents, APNS tokens, Firebase tokens, and bearer tokens must stay outside the repository.
